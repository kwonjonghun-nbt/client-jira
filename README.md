# Client Jira

Jira 데이터를 로컬에 수집하여 시각화하고, AI로 리포트와 이슈공유를 생성하는 Electron 데스크톱 앱.

## 어떤 문제를 해결하나요?

Jira에는 이슈 데이터가 풍부하지만, 팀 단위로 "지금 상황이 어떤지" 빠르게 파악하기 어렵습니다.

- 매일 아침 이슈공유를 위해 Jira 보드를 직접 훑어야 합니다
- 주간/월간 리포트를 쓸 때 이슈를 하나하나 정리해야 합니다
- 라벨별 완료율, 담당자별 워크로드 같은 통계를 보려면 별도 도구가 필요합니다
- 타임라인(간트 차트)을 보려면 유료 플러그인이 필요합니다
- OKR과 Jira 이슈를 연결하여 진행률을 추적할 방법이 없습니다

Client Jira는 Jira 데이터를 로컬에 동기화해두고, 대시보드/통계/타임라인/리포트를 한 곳에서 제공합니다. AI(Claude/Gemini)가 이슈를 분석하여 리포트와 이슈공유를 자동 생성합니다.

## 기능

### 대시보드

> **해결하는 문제**: "지금 우리 팀 상황이 어떤지" 한눈에 파악하기 어려움

- 요약 카드: 전체/진행중/완료/미착수 이슈 수
- 이번 주 마감 이슈 목록
- 담당자별 워크로드 바 차트
- 최근 업데이트 이슈, 이슈 타입별 분포, 변경 추적
- 기간/담당자 필터로 범위 조절

**사용법**: 사이드바에서 대시보드(🏠) 클릭. 기간 프리셋(1주~1년)이나 담당자 드롭다운으로 필터링합니다.

#### 오늘의 이슈공유

> **해결하는 문제**: 매일 아침 이슈공유를 준비하는 데 시간이 걸림

두 가지 생성 방식을 제공합니다:

- **데이터 기반 생성**: 담당자의 이슈를 진행/오늘마감/지연/리스크로 자동 분류하여 시각적 슬라이드로 표시합니다. 타임라인 Gantt, 티켓 카드, 요약 슬라이드로 구성되며, 키보드 좌우 키로 네비게이션합니다.
- **AI 이슈공유 생성**: Claude/Gemini가 이슈 데이터를 분석하여 이슈공유 마크다운을 생성합니다. 백그라운드에서 실행되므로 다른 페이지로 이동해도 됩니다.

**사용법**: 대시보드 내 "오늘의 이슈공유" 섹션에서 담당자 선택 후 생성 버튼 클릭.

### 과제 목록

> **해결하는 문제**: Jira 웹에서 여러 프로젝트의 이슈를 필터링하며 보기 불편

- 동기화된 전체 이슈를 테이블로 조회
- 프로젝트, 타입, 상태, 담당자, 우선순위 필터 + 키워드 검색
- 이슈 클릭 시 상세 모달: 전체 정보, 설명(Markdown 렌더링), 상태 전환 타임라인
- **티켓 분석하기**: AI가 해당 이슈를 분석하여 요약/리스크/권장사항을 제공

**사용법**: 사이드바에서 과제(📋) 클릭. 필터를 조합하여 원하는 이슈를 찾고, 행을 클릭하면 상세 모달이 열립니다. 상세 모달 하단의 "티켓 분석하기" 버튼으로 AI 분석을 실행합니다.

### 타임라인

> **해결하는 문제**: Jira에서 간트 차트를 보려면 유료 플러그인이 필요

- Jira 이슈를 간트 차트 형태로 시각화 (시작일 ~ 마감일)
- 일간/주간/월간 뷰 전환
- 줌 인/아웃, "오늘" 위치로 이동
- 이슈 타입별 바/행 표시 ON/OFF
- 오늘 날짜 기준선(빨간 수직선) 표시

**사용법**: 사이드바에서 타임라인(📅) 클릭. 우측 플로팅 패널에서 뷰 모드 전환, 줌 조절, "오늘" 버튼 사용. 필터를 적용하면 해당 이슈만 차트에 표시됩니다.

### 라벨별 통계

> **해결하는 문제**: 라벨 기준으로 작업 완료율을 파악하기 어려움

- 라벨별 전체/완료/미완료 이슈 수 및 완료율 프로그레스 바
- 테이블 뷰 / 도넛 차트 뷰 전환
- 기간 프리셋(7일/30일/90일/전체) + 커스텀 날짜 범위

**사용법**: 사이드바에서 통계(📊) 클릭. 기간 프리셋 버튼으로 빠르게 범위를 변경하거나, date picker로 직접 지정합니다. 우측 상단 테이블/차트 토글로 뷰 모드를 전환합니다.

### 라벨 메모

> **해결하는 문제**: 팀 내 라벨 컨벤션이 정리되어 있지 않음

- Jira 라벨의 의미와 용도를 기록
- 수집된 라벨 중 메모 미등록 항목 빠른 추가
- 500ms 디바운스 자동 저장

**사용법**: 사이드바에서 라벨 메모(🏷️) 클릭. Jira에서 수집된 라벨 중 미등록 항목이 상단에 표시되며, 클릭으로 추가합니다. 메모를 입력하면 자동 저장됩니다.

### 리포트

> **해결하는 문제**: 주간/월간 리포트 작성에 많은 시간이 소요됨

- 담당자, 기간 선택 후 **AI 리포트 생성** 버튼 클릭
- AI가 이슈 데이터 + 상태 전환 changelog를 분석하여 리포트 생성
  - 워크플로우 병목 구간 분석
  - 비정상 전환 패턴 감지 (역방향 전환, 중간 단계 건너뛰기, 형식적 상태 변경 등)
- 생성된 리포트를 `.md` 파일로 저장, 이후 조회 가능

**사용법**: 사이드바에서 리포트(📄) 클릭. 상단에서 담당자와 기간을 선택하고 "AI 리포트 생성" 클릭. AI가 백그라운드에서 분석하며, 완료 시 사이드바 🤖 뱃지에서 확인합니다. 결과를 확인하고 저장하면 리포트 목록에 추가됩니다.

### OKR

> **해결하는 문제**: OKR과 실제 Jira 작업 간의 연결고리가 없음

- Objective → Key Result 계층 관리
- KR에 Jira 이슈 연결 → 완료 비율로 KR 진행률 자동 계산 → Objective 진행률은 KR 평균
- 캔버스 모달: 카드 자유 배치, 그룹핑(서브그룹 지원), 카드 간 관계 연결 (직교 라우팅 화살표, 웨이포인트 편집)
- 가상 티켓 생성: Jira에 없는 작업도 캔버스에 추가 가능
- OKR 데이터 JSON 내보내기

**사용법**: 사이드바에서 OKR(🎯) 클릭. Objective를 추가하고, 하위에 KR을 생성합니다. KR을 클릭하면 캔버스 모달이 열리며, 여기서 Jira 이슈를 검색하여 연결합니다. 연결된 이슈의 완료 비율이 자동으로 KR 진행률에 반영됩니다.

### AI 태스크 매니저

> **해결하는 문제**: AI 분석/리포트 생성 중 다른 작업을 할 수 없음

모든 AI 작업(리포트, 이슈공유, 티켓 분석)은 백그라운드 태스크로 실행됩니다.

- 사이드바 하단 🤖 버튼의 뱃지로 상태 확인
  - 빨간 뱃지: 진행중인 작업 수
  - 초록 뱃지: 완료된 작업 수 (결과 확인 필요)
- 🤖 클릭 → 태스크 패널에서 진행률 확인, 실행 중 작업 중단 가능
- 완료된 태스크 클릭 시 결과 확인 및 저장

## 되는 것과 안 되는 것

### 되는 것

- Jira Cloud 프로젝트의 이슈 데이터 수집 및 시각화
- 여러 프로젝트를 한 번에 수집하여 통합 조회
- 자동 동기화 스케줄 설정 (node-cron 기반)
- Claude 또는 Gemini CLI를 활용한 AI 리포트/이슈공유/티켓 분석
- 오프라인 상태에서 마지막 동기화 데이터 조회 (로컬 JSON 저장)
- OKR 캔버스에서 Jira 이슈와 목표 연결, 진행률 자동 계산
- 앱 내 자동 업데이트 (GitHub Releases 기반)

### 안 되는 것

- **Jira에 쓰기 불가**: 이슈 상태 변경, 댓글 작성 등 Jira에 데이터를 쓰는 기능은 없습니다. 읽기 전용입니다.
- **Jira Server/Data Center 미지원**: Jira Cloud REST API v3만 지원합니다.
- **실시간 동기화 아님**: 스케줄 또는 수동 싱크 시점의 스냅샷입니다. Jira에서 즉시 반영되지 않습니다.
- **AI 기능은 CLI 설치 필요**: Claude CLI 또는 Gemini CLI가 시스템에 설치되어 있어야 합니다. AI 기능 없이도 데이터 시각화 기능은 모두 사용 가능합니다.
- **멀티 유저 아님**: 로컬 데스크톱 앱으로, 개인 사용 또는 화면 공유 용도입니다.
- **macOS 전용**: 현재 macOS용으로만 빌드됩니다.

## 초기 설정

### 1. 설치 및 실행

```bash
git clone git@github.com:kwonjonghun-nbt/client-jira.git
cd client-jira
npm install
npm start
```

### 2. Jira 연결

앱 실행 후 사이드바에서 **설정(⚙️)** 페이지로 이동합니다.

| 항목 | 설명 |
|------|------|
| Base URL | Jira 인스턴스 주소 (예: `https://your-team.atlassian.net`) |
| 이메일 | Jira 계정 이메일 |
| API 토큰 | [Atlassian API 토큰](https://id.atlassian.com/manage-profile/security/api-tokens)에서 생성 |

입력 후 **연결 테스트** 버튼으로 연결을 확인합니다. API 토큰은 시스템 키체인에 안전하게 저장됩니다.

### 3. 수집 대상 설정

- **프로젝트 선택**: 연결 테스트 성공 후, Jira에서 프로젝트 목록을 불러와 수집할 프로젝트를 선택합니다.
- **담당자 입력**: 수집 대상 담당자를 추가합니다. 비워두면 선택한 프로젝트의 전체 이슈를 수집합니다.

### 4. 첫 동기화

설정 완료 후 각 페이지 상단의 **싱크 버튼**을 눌러 첫 동기화를 실행합니다. 이슈 수에 따라 수 초 ~ 수 분이 소요되며, 진행률이 표시됩니다.

### 5. AI 기능 설정 (선택사항)

AI 리포트/이슈공유/티켓 분석 기능을 사용하려면:

1. **Claude CLI** 또는 **Gemini CLI**를 시스템에 설치합니다.
   - Claude: `npm install -g @anthropic-ai/claude-code`
   - Gemini: `npm install -g @google/gemini-cli`
2. 설치한 CLI를 터미널에서 한 번 실행하여 인증을 완료합니다 (최초 1회).
3. 설정 페이지의 **AI 에이전트** 섹션에서 사용할 CLI를 선택합니다.

> AI 기능은 선택사항입니다. 설치하지 않아도 대시보드, 과제 목록, 타임라인, 통계, OKR 등 모든 데이터 시각화 기능은 정상 작동합니다.

### 6. 자동 동기화 (선택사항)

설정 페이지의 **스케줄** 섹션에서 자동 동기화를 켜고 시간을 지정하면, 매일 해당 시각에 자동으로 데이터를 동기화합니다.
